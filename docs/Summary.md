# Отчёт о выполнении плана тестирования


## 1. Введение

Документ содержит итоги выполнения плана автоматизированного тестирования сервиса покупки тура в Марракеш, включая анализ выполненных работ и соблюдения временных рамок.

## 2. Запланированные и выполненные работы

**Запланировано:**
- Проведение автоматизированного тестирования сервиса покупки тура в Марракеш.
- Реализация позитивных и негативных сценариев тестирования.

**Выполнено:**
- Все запланированные работы выполнены в полном объёме.
- Протестированы позитивные и негативные сценарии взаимодействия с сервисом.
- Ни один из потенциально возможных рисков не реализовался в ходе выполнения работ.

## 3. Анализ рисков

В рамках планирования были идентифицированы потенциальные риски (в частности, изменение UI/UX и ломка локаторов).

**Итог по рискам:**
- Ни один из запланированных рисков **не сработал** в ходе выполнения тестирования.
- Стабильность интерфейса позволила избежать проблем с локаторами.
- Автотесты выполнялись корректно на протяжении всего цикла тестирования.

## 4. Хронологические рамки и выполнение сроков

План работ включал следующие этапы:

1. **Планирование автоматизации тестирования**
    - Планируемый срок: до 05.12.2025 г.
    - Фактическое выполнение: выполнено раньше срока.

2. **Настройка окружения, написание и отладка автотестов**
    - Планируемый срок: до 10.12.2025 г.
    - Фактическое выполнение: выполнено раньше срока.

3. **Подготовка отчётных документов по итогам автоматизированного тестирования**
    - Планируемый срок: до 13.12.2025 г.
    - Фактическое выполнение: выполнено раньше срока.

4. **Подготовка отчётных документов по итогам автоматизации**
    - Планируемый срок: до 17.12.2025 г.
    - Фактическое выполнение: выполнено раньше срока.

**Общий вывод по срокам:** все этапы завершены **раньше запланированного срока**.

## 5. Итоги и выводы

- План тестирования реализован полностью и без инцидентов.
- Отсутствие реализовавшихся рисков подтверждает корректность предварительной оценки и мер предосторожности.
- Соблюдение и опережение сроков свидетельствует об эффективной организации процесса автоматизации тестирования.
- Стабильная работа автотестов подтверждает качество подготовки тестовой базы и выбор стратегии локализации элементов.


## 6. Рекомендации

1. В целях повышения устойчивости автотестов к изменениям интерфейса внедрить использование атрибутов `data-test-id` в качестве основных локаторов. Для всех ключевых элементов интерфейса:
   - добавить атрибуты `data-test-id` с понятными, уникальными значениями;
   - переработать существующие автотесты для использования новых локаторов;
   - закрепить правило добавления `data-test-id` при разработке новых элементов интерфейса.
2. Продолжить мониторинг стабильности интерфейса в будущих циклах тестирования.
3. Зафиксировать успешный опыт для использования в аналогичных проектах.
4. Проводить регулярный аудит автотестов для поддержания их актуальности.
